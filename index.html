Flagship Case 01 · AI Agent

EVA — From fragmented work signals to confirmable AI drafts.

A multi-module AI agent for knowledge workers. EVA listens across meetings, chat, email, tasks, and travel — and turns the noise into drafts the user can confirm, edit, or reject.

Role
Senior Product Designer · lead
Scope
Phase 2 · five modules · knowledge-graph substrate
Design thesis
AI proposes. Human disposes.
Key contribution
Human-in-the-loop framework across modules
Problem

Knowledge workers don't lack tools. They lack a layer above them.

Meetings, IM, email, tasks, and travel each have a category leader — but the cognitive load lives in the seams between them. Action items get lost between a call and a chat thread. Existing assistants answer questions on demand; the opportunity is an agent that maintains context proactively, while keeping the human in firm control of every commit.

Experience Model

Signal → Interpretation → Draft → Confirmation → Action

The same five-stage loop sits underneath every module — so users learn the shape of the agent once.

01
Signal
Inbound from meetings, chat, email, calendar — the agent listens, never asks the user to teach it.

02

→

→
04

Confirm, edit, ignore — every artefact ships with the same three controls and a reasoning string.
→
05
Action
Only on confirm. Reversible. Source-linked. Logged.

Design Principles

Three principles the whole product is bound to.

01

Drafts, not commits

Every model output lands as an editable draft — agenda, task, email, plan. Nothing is finalized without an explicit confirm.

02

Explain before asking for trust

Each suggestion ships with a short "why" string. Users can override at any time; the system treats overrides as ground truth.

03

Always reversible

Created items remain editable. Reminders are dismissible. Drafts can be discarded. The agent is never the last word.

What this case demonstrates — Designing AI behaviour as a product surface · structuring multi-module agent experiences · turning model outputs into trustworthy drafts · transferable patterns for AI inside operational and enterprise products.
